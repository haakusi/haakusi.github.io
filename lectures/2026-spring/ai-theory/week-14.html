Week 14

2026-06-01 - 06-07 · AI Games, Minimax, and Alpha-Beta Pruning

Adversarial Search

교안: AI and Game.pdf, Minimax.pdf, Alpha_beta_prunning.pdf (AIMA 5장) · 핵심 개념 중심의 개인 요약

14주차는 게임을 상대의 합리적 대응까지 고려하는 탐색 문제로 바라봤다. 게임 상태를 트리로 펼친 뒤 MAX와 MIN의 선택을 번갈아 계산하는 Minimax, 현실적인 자원 한계를 다루는 깊이 제한과 평가함수, 결과를 바꾸지 않으면서 불필요한 가지를 제거하는 알파-베타 가지치기가 핵심이다.

핵심 관점. 일반 탐색에서는 환경이 수동적이지만, 게임에서는 상대가 내 목표를 방해하도록 움직인다. 따라서 좋은 수는 당장 점수가 높은 수가 아니라, 상대의 최선 대응 이후에도 가장 나은 결과를 보장하는 수다.

1. 게임을 탐색 문제로 표현하기

구성 요소 질문 역할
Initial state 어디서 시작하는가? 초기 보드와 첫 플레이어를 정의한다.
Player and Actions 누가 어떤 수를 둘 수 있는가? 현재 차례와 합법 행동을 정한다.
Result 한 수 뒤 상태는 무엇인가? 행동에 따른 상태 전이를 만든다.
Terminal and Utility 언제 끝나며 결과는 얼마인가? 승패나 점수를 수치 효용으로 바꾼다.

결정적·확률적, 완전·불완전 정보, 1인·2인·다인, 순차·동시 행동, 제로섬 여부에 따라 필요한 알고리즘이 달라진다. 이 주차의 표준 모델은 결정적이고 완전히 관찰 가능하며, 두 플레이어가 번갈아 움직이는 제로섬 게임이다.

2. Minimax: 최선의 상대를 가정하기

게임 트리에서 MAX는 루트 플레이어의 효용을 최대화하고, MIN은 그 효용을 최소화한다. 말단 상태의 효용을 위로 전달하면, 각 MAX 노드는 자식의 최댓값을, 각 MIN 노드는 자식의 최솟값을 선택한다.

terminal: V(s) = Utility(s)
MAX turn: V(s) = max V(Result(s, a))
MIN turn: V(s) = min V(Result(s, a))

Complete

유한한 게임 트리라면 말단까지 탐색해 답을 찾는다.

Optimal

상대도 최적으로 둔다는 가정 아래 최선의 보장값을 선택한다.

Cost

시간은 O(b^m), 깊이 우선 구현의 공간은 O(bm)으로 커진다.

3. 자원 한계: 깊이 제한과 평가함수

현실적인 게임은 분기 계수와 깊이가 커서 말단까지 볼 수 없다. Bounded lookahead는 정해진 깊이에서 탐색을 멈추고, 비말단 상태를 평가함수로 점수화한다. 더 깊은 탐색은 보통 더 좋은 결정을 주지만 계산량도 급격히 늘어난다.

Evaluation(s) = w1 f1(s) + w2 f2(s) + ... + wn fn(s)
특징은 위치의 강점 · 가중치는 각 특징의 상대적 중요도

평가함수는 실제 minimax 값을 근사하므로 최적성 보장은 약해진다. 대신 제한된 시간 안에서 탐색 깊이와 특징 설계 사이의 균형을 잡아 충분히 좋은 결정을 만든다.

4. 알파-베타 가지치기

경계 의미 가지치기 조건
alpha 현재 경로에서 MAX가 이미 확보한 최선의 하한 MIN 노드의 값이 alpha 이하이면 나머지 자식을 볼 필요가 없다.
beta 현재 경로에서 MIN이 이미 확보한 최선의 상한 MAX 노드의 값이 beta 이상이면 나머지 자식을 볼 필요가 없다.

이미 더 좋은 대안이 조상 노드에 존재한다면 현재 가지의 남은 결과는 최종 선택을 바꿀 수 없다. 이때 탐색을 중단해도 Minimax가 선택하는 값과 행동은 그대로 유지된다.

5. 같은 답을 더 적은 탐색으로

알파-베타의 효과는 수의 탐색 순서에 크게 의존한다. 좋은 수를 먼저 보면 경계가 빨리 좁아져 더 많은 가지를 제거한다. 완벽한 순서에서는 시간복잡도가 O(b^(m/2)) 수준으로 줄어, 같은 계산 예산으로 대략 두 배 깊이까지 볼 수 있다.

연결. Minimax가 상대를 포함한 의사결정 원리를 정의한다면, 평가함수는 제한된 깊이에서 상태의 가치를 근사하고, 알파-베타는 답에 영향을 주지 않는 계산을 생략한다.

6. 주차 요약

  • 게임 탐색은 내 행동뿐 아니라 상대의 최선 대응까지 포함한다.
  • Minimax는 MAX의 최대화와 MIN의 최소화를 번갈아 계산한다.
  • 큰 게임에서는 깊이 제한과 평가함수로 비말단 상태를 근사한다.
  • 알파-베타는 최종 Minimax 결과를 유지하면서 불필요한 가지를 제거한다.
  • 좋은 수를 먼저 검사할수록 가지치기가 커지고 더 깊은 탐색이 가능해진다.

Sources: AI and Game.pdf, Minimax.pdf, and Alpha_beta_prunning.pdf (AIMA Chapter 5) · Personal summary centered on key concepts

Week 14 treated games as search problems in which an opponent actively responds. The central sequence was to unfold states into a game tree, alternate MAX and MIN choices through minimax, cope with resource limits through depth bounds and evaluation functions, and skip irrelevant branches with alpha-beta pruning.

Core perspective. In ordinary search the environment may be passive. In a game, the opponent chooses actions that work against the root player's goal. A strong move is therefore not merely attractive now; it must remain strong after the opponent's best reply.

1. Representing a Game as Search

Component Question Role
Initial state Where does play begin? Defines the initial board and first player.
Player and Actions Who moves, and which moves are legal? Identifies the turn and available actions.
Result What state follows a move? Defines the state transition.
Terminal and Utility When does play end, and what is the outcome? Maps a win, loss, or score to numeric utility.

Algorithms differ according to whether a game is deterministic or stochastic, fully or partially observable, single-player or multi-player, sequential or simultaneous, and zero-sum or general-sum. This week's standard model was a deterministic, fully observable, turn-taking, two-player zero-sum game.

2. Minimax: Assuming the Best Opponent

In a game tree, MAX maximizes the root player's utility and MIN minimizes it. Terminal utilities are propagated upward: every MAX node selects its largest child value, while every MIN node selects its smallest child value.

terminal: V(s) = Utility(s)
MAX turn: V(s) = max V(Result(s, a))
MIN turn: V(s) = min V(Result(s, a))

Complete

On a finite game tree, search can reach an answer.

Optimal

Against optimal opposition, it selects the best guaranteed value.

Cost

Time grows as O(b^m), while depth-first space is O(bm).

3. Resource Limits: Depth Bounds and Evaluation

Real games have branching factors and depths too large for terminal search. Bounded lookahead stops at a selected depth and scores non-terminal positions with an evaluation function. Deeper search usually improves decisions, but its cost rises rapidly.

Evaluation(s) = w1 f1(s) + w2 f2(s) + ... + wn fn(s)
features describe positional strength · weights encode relative importance

An evaluation function approximates the true minimax value, so the guarantee of perfect play is weakened. In return, the agent balances search depth and feature quality to make a useful decision within a fixed budget.

4. Alpha-Beta Pruning

Bound Meaning Cutoff
alpha The best lower bound MAX has secured along the path If a MIN node falls to alpha or below, its remaining children cannot matter.
beta The best upper bound MIN has secured along the path If a MAX node reaches beta or above, its remaining children cannot matter.

When an ancestor already has a better alternative, the rest of the current branch cannot change the final choice. Stopping that search preserves both the minimax value and the selected action.

5. The Same Answer with Less Search

Alpha-beta performance depends strongly on move ordering. Examining strong moves first tightens the bounds earlier and creates more cutoffs. With perfect ordering, time approaches O(b^(m/2)), allowing roughly twice the depth under the same search budget.

Connection. Minimax defines the decision principle under opposition, the evaluation function estimates value at a limited depth, and alpha-beta removes work that cannot influence the decision.

6. Week Summary

  • Game search includes the opponent's best response, not just the agent's action.
  • Minimax alternates MAX's maximization with MIN's minimization.
  • Large games require depth limits and evaluation functions for non-terminal states.
  • Alpha-beta removes irrelevant branches without changing the final minimax result.
  • Better move ordering creates more cutoffs and enables deeper search.
