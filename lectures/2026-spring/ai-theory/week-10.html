1. 발표 주제: 머신 언러닝과 LLM의 망각

머신 언러닝은 학습된 모델에서 특정 정보나 학습 흔적을 제거하거나 약화하는 문제로 볼 수 있다. 발표에서는 LLM의 망각을 "조금 지우는 것"이라는 표현으로 생각해볼 수 있었다. 단순히 데이터를 삭제하는 문제가 아니라, 이미 모델 내부의 가중치와 행동 경향에 반영된 내용을 어떻게 다룰 것인가의 문제다.

관점 정리
Machine Unlearning 모델이 이미 학습한 특정 정보나 패턴을 제거하거나 완화하려는 접근
LLM의 망각 특정 기억을 단순 파일처럼 지우는 것이 아니라, 모델 행동에 남은 흔적을 어떻게 낮출지 고민하는 문제
발표 후 생각 망각은 안전성, 프라이버시, 모델 수정, 서비스 운영과 연결될 수 있다

2. Reward Function을 고치는 행위

발표를 통해 reward function을 고치는 행위가 자율적인 요소를 가진 시스템에서 매우 중요한 요소가 될 수 있겠다는 생각이 남았다. 에이전트가 스스로 판단하고 행동하는 구조에서는 어떤 행동을 보상하고 어떤 행동을 피하게 할 것인지가 전체 시스템의 방향을 바꿀 수 있다.

자율적 시스템
-> 목표 설정
-> reward function
-> 행동 선택과 수정
-> 결과와 피드백

따라서 reward function은 단순한 점수 함수가 아니라, 에이전트가 어떤 행동을 더 가치 있게 여기도록 만들 것인가의 설계 지점이다. 머신 언러닝이 모델의 기억과 행동을 수정하는 문제라면, reward function의 수정은 앞으로의 행동 경향을 조정하는 문제로 연결해 볼 수 있다.

3. 수업 실습: A* 탐색 구현

A*는 목표까지의 실제 비용과 앞으로 남은 비용의 추정치를 함께 사용해 탐색하는 알고리즘이다. BFS가 깊이 순서대로 넓게 확장한다면, A*는 우선순위 큐를 사용해 더 유망해 보이는 노드를 먼저 확장한다. 핵심 평가는 다음과 같이 정리할 수 있다.

f(n) = g(n) + h(n)
g(n): 시작점에서 현재 노드까지 실제로 든 비용
h(n): 현재 노드에서 목표까지 남은 비용의 추정치
A* 구현 요소 역할
Open set 앞으로 확장할 후보 노드 집합. 보통 우선순위 큐로 관리한다.
Closed / visited set 이미 처리했거나 더 좋은 경로를 찾은 노드를 관리해 중복 확장을 줄인다.
g score 각 노드까지 도달하는 현재까지의 최저 비용을 기록한다.
heuristic 목표까지 남은 거리를 추정한다. 이 값이 탐색의 방향성을 만든다.
came from 최종 경로를 복원하기 위해 각 노드의 이전 노드를 기록한다.

좋은 heuristic은 탐색 공간을 줄이는 데 도움을 준다. 다만 heuristic이 실제 남은 비용을 과하게 낮추거나 높이는 방식에 따라 탐색 결과와 최적성 판단이 달라질 수 있다. 일반적으로 admissible heuristic, 즉 실제 비용을 과대평가하지 않는 휴리스틱을 사용하면 A*의 최적성 논의가 가능해진다.

4. BFS와 A*의 핵심 비교

BFS와 A*는 모두 상태 공간을 탐색하지만, 어떤 노드를 먼저 볼 것인지가 다르다. BFS는 시작점에서 가까운 깊이부터 차례대로 확장하고, A*는 현재까지의 비용과 목표까지의 추정 비용을 함께 고려해 우선순위를 정한다.

구분 BFS A*
자료구조 FIFO queue Priority queue
확장 기준 깊이가 얕은 노드부터 확장 f(n) = g(n) + h(n) 값이 작은 노드부터 확장
휴리스틱 사용하지 않음 사용함
장점 가중치가 같거나 없는 그래프에서 최단 경로를 단순하게 보장하기 쉽다 좋은 휴리스틱이 있으면 목표 방향으로 더 효율적으로 탐색할 수 있다
주의점 분기 수가 크면 너무 많은 노드를 확장할 수 있다 휴리스틱과 방문 처리, 비용 갱신을 잘못 구현하면 잘못된 경로를 선택할 수 있다

수업에서 A*를 직접 구현해본 이유는 BFS와 비교했을 때 "정보가 있는 탐색"이 어떻게 달라지는지 보기 위해서다. BFS는 문제의 구조를 모를 때도 넓게 탐색하지만, A*는 목표에 가까워 보이는 방향을 선택한다. 그래서 구현에서는 heuristic을 어떻게 정의하는지, 우선순위 큐에서 어떤 노드를 꺼내는지, 이미 방문한 노드에 더 좋은 비용이 발견되었을 때 어떻게 갱신하는지가 중요하다.

5. 첫 번째 과제 피드백: BFS/DFS 구현과 복잡도

첫 번째 과제에서는 각자가 BFS와 DFS를 다르게 구현했고, 그 차이를 바탕으로 fail case와 복잡도 케이스를 살펴보았다. 같은 문제를 풀어도 자료구조를 어떻게 쓰는지, goal test를 어디에 두는지, 반복문 구조를 어떻게 잡는지에 따라 실행 흐름이 달라진다.

피드백 지점 수업 메모
구현 방식의 차이 각자 BFS/DFS를 다르게 구현했고, 그 차이 자체가 비교 대상이 되었다
해시맵 사용 복잡도 측면에서 해시맵을 어떻게 쓰는지, 불필요한 중복 탐색을 줄이는지가 중요했다
복잡도 질문 구현 방식에 따라 복잡도가 크게 증가할 수 있고, 경우에 따라 높은 차수까지 갈 수 있는지 논의했다
fail case 정상 케이스뿐 아니라 실패 케이스에서 알고리즘이 어떻게 멈추고 판단하는지도 확인해야 한다

6. Goal Test의 위치

goal test를 어디에 배치하는지도 중요한 피드백이었다. 수업 메모상 goal test가 가장 상단에 있을 때와, 루프 아래로 들어갈 때 복잡도 표현이 달라질 수 있다고 정리되었다.

상단에서 goal test

목표 상태를 먼저 확인하면 불필요한 확장을 줄일 수 있다. 수업 메모에서는 O(b^d) 수준으로 비교되었다.

루프 아래에서 goal test

목표 확인이 늦어지면 한 단계 더 확장한 뒤 검사하게 될 수 있다. 수업 메모에서는 O(b^(d+1)) 수준으로 비교되었다.

이 차이는 단순히 코드 위치의 문제가 아니라, 탐색 알고리즘에서 언제 상태를 검사하고 언제 자식을 생성하는지의 문제다. 같은 BFS/DFS라도 goal test 위치와 방문 처리 방식에 따라 불필요한 노드 확장이 생길 수 있다.

1. Machine Unlearning and LLM Forgetting

Machine unlearning can be understood as an attempt to remove or reduce specific information or patterns from a trained model. In the presentation, LLM forgetting was framed as a problem of partially erasing or weakening a learned trace. This is not the same as deleting a file, because the information may already be reflected in weights and behavioral tendencies.

Lens Summary
Machine unlearning Reducing or removing selected learned information or patterns from a model
LLM forgetting Controlling traces that remain in model behavior rather than deleting a simple record
Follow-up thought Forgetting connects to safety, privacy, model updates, and service operation

2. Reward Function as a Control Point

The discussion also suggested that changing a reward function can be important in systems with autonomous behavior. When an agent judges and acts by itself, the reward function affects which behaviors are encouraged and which are discouraged.

autonomous system
-> goal setting
-> reward function
-> action selection and adjustment
-> outcome and feedback

In that sense, a reward function is not merely a scoring formula. It is a design point that shapes what the agent treats as valuable. If machine unlearning adjusts memory or behavior already learned, reward function modification can be seen as adjusting future behavioral tendencies.

3. In-Class Practice: A* Search

A* combines the actual cost from the start node with an estimate of the remaining cost to the goal. BFS expands broadly by depth, while A* uses a priority queue to expand the node that looks most promising.

f(n) = g(n) + h(n)
g(n): actual cost from the start to the current node
h(n): estimated remaining cost from the current node to the goal
A* implementation element Role
Open set Candidate nodes to expand, usually managed with a priority queue
Closed / visited set Reduces repeated expansion by tracking processed nodes or better-known paths
g score Stores the best known cost to reach each node
Heuristic Estimates remaining distance to the goal and gives the search direction
Came from Stores predecessor nodes so the final path can be reconstructed

A useful heuristic can reduce the search space. However, the result and optimality discussion depend on how the heuristic estimates the remaining cost. With an admissible heuristic, meaning one that does not overestimate the true remaining cost, A* can be discussed as an optimal search method under the usual assumptions.

4. BFS and A* Core Comparison

BFS and A* both search a state space, but they differ in which node is expanded first. BFS expands by shallowest depth. A* combines the cost so far with a heuristic estimate of the remaining cost.

Aspect BFS A*
Data structure FIFO queue Priority queue
Expansion rule Expand shallower nodes first Expand nodes with lower f(n) = g(n) + h(n)
Heuristic Not used Used
Strength Simple shortest-path behavior in unweighted or equal-cost graphs Can search more efficiently toward the goal when the heuristic is useful
Risk Can expand too many nodes when the branching factor is large Incorrect heuristic, visit handling, or cost update can lead to wrong behavior

The point of implementing A* in class was to see how informed search differs from BFS. BFS explores broadly without domain guidance, while A* uses an estimate of the goal direction. Implementation details such as heuristic definition, priority queue ordering, and updating a node when a better cost is found are therefore critical.

6. Goal Test Placement

Goal test placement was another important feedback point. The class note compared putting the goal test near the top of the loop with putting it below expansion logic.

Goal test first

Checking the goal earlier can reduce unnecessary expansion. The class note compared this with O(b^d).

Goal test after expansion

Checking the goal later can cause one more level of expansion before the test. The class note compared this with O(b^(d+1)).

The point is not only code placement. It is about when a search algorithm tests a state and when it generates children. In BFS/DFS, goal test placement and visited-state handling can create unnecessary node expansion.

7. Problem and Goal Structure

The feedback also noted that implementing Problem and Goal in the structure used by the textbook was helpful. Separating problem definition, goal condition, and search procedure makes the algorithm easier to inspect, especially when checking failure cases and complexity.

Structure Meaning
Problem Initial state, possible actions, state transitions, and related problem definition
Goal The condition used to decide whether the current state is a target state
Search The procedure that explores the state space using the problem and goal definitions
