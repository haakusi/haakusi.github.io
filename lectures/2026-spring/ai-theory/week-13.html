Constraint Satisfaction Problems

교안: 06. CSP-1.pdf, 06. CSP-2.pdf, CSP (3) 교안.pdf (AIMA 6장) · 핵심 개념 중심의 개인 요약

13주차는 제약 만족 문제(Constraint Satisfaction Problem, CSP)를 중심으로, 문제를 변수·도메인·제약으로 표현하고 해를 찾는 과정을 정리했다. 기본 백트래킹에서 출발해 MRV와 LCV 같은 순서 휴리스틱, forward checking과 AC-3 제약 전파, min-conflicts 지역 탐색까지 이어지는 흐름이다.

핵심 관점. 일반 탐색이 상태를 블랙박스처럼 다루는 데 비해 CSP는 상태의 내부를 변수와 값의 할당으로 드러낸다. 해는 모든 변수가 할당된 complete assignment이면서 모든 제약을 만족하는 consistent assignment이어야 한다.

1. CSP의 형식적 표현

CSP = (X, D, C)
X = 변수 집합 · D = 각 변수의 도메인 · C = 허용되는 할당을 정의하는 제약 집합
Goal = complete assignment + consistent assignment
요소 의미 예시
Variable 값을 결정해야 하는 대상 지역, 스도쿠 칸, 강의, 문자
Domain 각 변수가 가질 수 있는 값의 집합 색상, 1-9, 시간·강의실, 0-9
Constraint 허용되거나 금지되는 값의 조합 인접 지역은 다른 색, 행·열은 AllDifferent

교안은 지도 색칠, 스도쿠, N-Queens, 복면산, 강의실 배정을 대표 예제로 사용한다. 문제의 표면은 서로 다르지만, 모두 어떤 값을 정해야 하는 변수와 가능한 값의 범위, 동시에 만족해야 하는 규칙으로 다시 쓸 수 있다.

2. 백트래킹: 한 번에 변수 하나씩

백트래킹은 깊이 우선 탐색을 CSP에 맞게 단순화한 기본 알고리즘이다. 빈 할당에서 시작해 미할당 변수 하나를 고르고, 현재 할당과 충돌하지 않는 값을 넣은 뒤 재귀적으로 다음 변수를 처리한다. 합법적인 값이 남지 않으면 직전 선택을 취소하고 다른 값을 시도한다.

BACKTRACK(assignment, csp):
    if assignment is complete:
        return assignment

    var = SELECT-UNASSIGNED-VARIABLE(csp)
    for value in ORDER-DOMAIN-VALUES(var, assignment, csp):
        if value is consistent with assignment:
            assign var = value
            result = BACKTRACK(assignment, csp)
            if result is not failure:
                return result
            undo var = value

    return failure

변수 할당은 순서를 바꾸어도 같은 부분 할당을 나타내므로, 매 노드에서 변수 하나만 선택하면 된다. 또한 값을 넣을 때마다 제약을 검사하면 이미 모순인 부분 할당의 하위 탐색을 즉시 버릴 수 있다.

3. 더 빨리 실패하고 더 빨리 성공하는 순서

MRV

합법적인 값이 가장 적게 남은 변수를 먼저 고른다. 실패 가능성이 큰 지점을 일찍 확인하는 fail-fast 전략이다.

Degree Heuristic

MRV가 동률이면 아직 할당되지 않은 변수들과 가장 많은 제약을 공유하는 변수를 우선한다.

LCV

이웃 변수의 도메인에서 가장 적은 값을 제거하는 값을 먼저 시도한다. 이후 선택지를 최대한 남기는 succeed-fast 전략이다.

4. Forward Checking과 AC-3

방법 확인 범위 역할
Forward checking 방금 할당한 변수의 미할당 이웃 충돌하는 값을 이웃 도메인에서 지우고, 빈 도메인이 생기면 즉시 백트래킹한다.
Arc consistency 서로 연결된 변수 쌍 X의 모든 값이 Y에서 최소 하나의 양립 가능한 값을 갖도록 도메인을 줄인다.
AC-3 큐에 들어 있는 모든 arc 도메인이 바뀔 때 이웃 arc를 다시 검사해 제약 축소를 연쇄적으로 전파한다.
Forward checking = 한 단계 앞의 명백한 실패 감지
AC-3 = 이웃 도메인 축소를 반복 전파 · 시간복잡도 O(c * d^3)

Forward checking은 현재 변수와 이웃 사이의 충돌은 잡지만, 아직 할당되지 않은 이웃들끼리의 모순은 놓칠 수 있다. Arc consistency는 한 도메인의 값이 다른 도메인에서 지원되는지 반복 확인해 더 이른 실패 감지를 가능하게 한다.

5. Min-Conflicts와 지역 탐색

CSP를 항상 부분 할당으로만 풀 필요는 없다. 지역 탐색은 처음부터 모든 변수에 값을 넣은 complete-state formulation을 사용하고, 위반된 제약의 수를 h(n)으로 둔다. Min-conflicts는 충돌 중인 변수를 하나 고른 뒤 제약 위반을 가장 적게 만드는 값으로 바꾸며 h(n) = 0인 상태를 찾는다.

12주차와의 연결. CSP 교안은 hill climbing, simulated annealing, genetic algorithm도 complete-state CSP 탐색에 사용할 수 있다고 설명한다. 따라서 앞선 좀비 생존 키트 GA 실습은 제약 아래에서 더 좋은 해를 찾는 또 하나의 탐색 관점으로 이어진다.

6. 주차 요약

  • CSP는 문제를 변수, 도메인, 제약으로 명시적으로 표현한다.
  • 백트래킹은 변수 하나씩 값을 넣고 모순이 생기면 선택을 되돌린다.
  • MRV와 degree heuristic은 변수 순서를, LCV는 값의 순서를 개선한다.
  • Forward checking과 AC-3는 미래의 모순을 더 일찍 발견하도록 도메인을 축소한다.
  • Min-conflicts와 GA 같은 지역 탐색은 complete-state에서 제약 위반을 줄이는 대안적 접근이다.

Week 13 focused on Constraint Satisfaction Problems (CSPs): representing a problem through variables, domains, and constraints, then searching for a valid assignment. The sequence moved from basic backtracking to ordering heuristics such as MRV and LCV, constraint propagation through forward checking and AC-3, and min-conflicts local search.

Core perspective. Standard search may treat a state as a black box. A CSP exposes its internal structure as variable-value assignments. A solution must be both a complete assignment, with every variable assigned, and a consistent assignment, satisfying every constraint.

1. Formal CSP Representation

CSP = (X, D, C)
X = variables · D = each variable's domain · C = constraints that define legal assignments
Goal = complete assignment + consistent assignment
Element Meaning Examples
Variable An object whose value must be chosen Region, Sudoku cell, class, letter
Domain The set of values available to a variable Colors, 1-9, time and room, 0-9
Constraint A legal or forbidden combination of values Adjacent regions differ; each row and column is AllDifferent

The handouts use map coloring, Sudoku, N-Queens, cryptarithmetic, and classroom assignment as representative examples. Their surfaces differ, but each can be rewritten as variables to decide, possible values, and rules that must hold together.

2. Backtracking: One Variable at a Time

Backtracking is the fundamental depth-first algorithm adapted to CSP structure. It begins with an empty assignment, selects one unassigned variable, tries a value consistent with the current assignment, and recursively continues. If no legal value remains, it undoes the previous choice and tries another value.

BACKTRACK(assignment, csp):
    if assignment is complete:
        return assignment

    var = SELECT-UNASSIGNED-VARIABLE(csp)
    for value in ORDER-DOMAIN-VALUES(var, assignment, csp):
        if value is consistent with assignment:
            assign var = value
            result = BACKTRACK(assignment, csp)
            if result is not failure:
                return result
            undo var = value

    return failure

Variable assignments are commutative, so the algorithm only needs to select one variable at each node. Checking constraints as values are assigned also lets it discard inconsistent partial assignments before exploring their descendants.

3. Ordering to Fail Fast and Succeed Fast

MRV

Choose the variable with the fewest legal values remaining. It exposes likely failure points early.

Degree Heuristic

Break an MRV tie by choosing the variable involved in the most constraints with unassigned variables.

LCV

Try the value that removes the fewest values from neighboring domains, preserving more future choices.

4. Forward Checking and AC-3

Method Scope Role
Forward checking Unassigned neighbors of the newly assigned variable Remove conflicting neighbor values and backtrack if a domain becomes empty.
Arc consistency Pairs of connected variables Keep each value of X only when Y contains at least one compatible value.
AC-3 All arcs managed in a queue When a domain changes, recheck neighboring arcs and propagate the reduction.
Forward checking = detect obvious failure one step ahead
AC-3 = repeatedly propagate neighbor-domain reductions · time complexity O(c * d^3)

Forward checking detects conflicts between the current variable and its neighbors, but it can miss contradictions among still-unassigned neighbors. Arc consistency repeatedly checks whether values have support in neighboring domains, allowing earlier failure detection.

5. Min-Conflicts and Local Search

A CSP does not always require partial assignments. Local search uses a complete-state formulation in which every variable has a value from the beginning and h(n) is the number of violated constraints. Min-conflicts selects a conflicted variable and assigns the value that creates the fewest violations until it reaches h(n) = 0.

Connection to Week 12. The CSP handout states that hill climbing, simulated annealing, and genetic algorithms can also use a complete-state CSP formulation. The earlier Zombie Survival Kit GA practice therefore provides another perspective on searching for stronger solutions under constraints.

6. Week Summary

  • A CSP represents a problem explicitly through variables, domains, and constraints.
  • Backtracking assigns one variable at a time and undoes a choice when it reaches inconsistency.
  • MRV and the degree heuristic order variables; LCV orders values.
  • Forward checking and AC-3 reduce domains to reveal future contradictions earlier.
  • Min-conflicts and local methods such as GAs offer complete-state alternatives.
