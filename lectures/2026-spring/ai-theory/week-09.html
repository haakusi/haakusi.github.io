Introduction to AI

Keywords: artificial intelligence, intelligence, Turing Test, Chinese Room, weak AI, strong AI, AGI, singularity, symbolism, connectionism, LLM agent, ReAct, CoT, ToT, memory, tools, MCP, AI safety

인공지능론은 9주차가 첫 수업이었다. 이 과목은 두 달 동안 온라인 3시간과 오프라인 3시간을 병행하는 압축형 수업이고, 첫 시간에는 인공지능의 고전적 질문부터 LLM 에이전트까지 큰 지도를 훑었다. 알고리즘, 정렬, 그래프, 지식그래프, 심볼릭 AI, 에이전트, 메모리, 도구, MCP, 그리고 앞으로의 직업 변화까지 한 번에 연결하는 오리엔테이션에 가까웠다.

메모. 이번 수업의 핵심은 "AI가 무엇인가"를 하나의 정의로 외우는 것이 아니라, 인간의 지능을 어떻게 흉내 내고, 문제 해결을 어떻게 인위적으로 구성하며, LLM 이후의 에이전트 시스템이 왜 다시 고전 AI의 개념을 필요로 하는지 보는 것이었다.

1. Artificial Intelligence is...

수업은 인공지능을 두 가지 방향에서 출발시켰다. 하나는 인간의 지능을 흉내 내는 것이고, 다른 하나는 인위적으로 만든 문제 해결 시스템이다. 지능은 오감으로 환경을 지각하고, 학습하고, 지식을 축적한 뒤 그것을 활용해 문제를 해결하는 능력으로 볼 수 있다.

개념 수업에서의 정리 중요한 이유
Artificial Intelligence 인간 지능의 흉내이자 인위적 문제 해결 모방과 문제 해결이라는 두 축이 모두 필요하다
Intelligence 지각, 학습, 지식, 활용의 결합 보는 것, 아는 것, 쓰는 것은 같은 일이 아니다
Perception 환경을 받아들이는 과정 사람도 집중 상태에 따라 보지 못하는 것이 생긴다

흥미로웠던 지점은 사람의 지각도 완전하지 않다는 것이다. 사람들은 "지각했다"를 "다 봤다"로 생각하기 쉽지만, 실제로는 집중하면 놓치는 것이 생긴다. 따라서 AI를 이해하려면 기계가 사람처럼 생각하는가만 볼 것이 아니라, 사람의 생각과 의미 도출 방식 자체도 아직 완벽히 설명되지 않았다는 점을 함께 봐야 한다.

2. Turing Test와 Chinese Room

튜링 테스트는 Alan Turing의 "Computing Machinery and Intelligence"에서 출발한 질문이다. 내부를 들여다보지 않고 결과만 보고 기계가 지능적으로 보이는지 판단하자는 접근이다. 단순해 보이지만 70년 가까이 유효했던 이유는, 지능을 직접 정의하기보다 행동과 결과로 판단한다는 실용적 태도를 갖고 있기 때문이다.

Turing Test

내부 구조를 보지 않고 대화 결과만으로 지능처럼 보이는지를 판단한다. 환경을 어떻게 세팅하느냐에 따라 결과가 달라질 수 있다.

Chinese Room

중국어를 모르는 사람이 규칙표만 보고 답을 만들 수 있다면, 그 방은 정말 중국어를 이해한다고 볼 수 있는가를 묻는다.

Chinese Room Thought Experiment는 튜링 테스트식 판단에 대한 반론으로 소개되었다. 겉으로 맞는 답을 내는 것과 실제로 이해하는 것은 다르다는 주장이다. 반대로 시스템 전체를 하나의 기계로 보면, 방 안의 사람만이 아니라 방 전체가 중국어를 처리하는 시스템이라고 볼 수도 있다. GPT 시대에는 이 논쟁이 다시 중요해졌다. 모델의 출력이 맞는지, 그 출력이 이해인지, 우리는 어떤 기준으로 반박하고 검증할지 스스로 생각해야 한다.

Eugene Goostman 사례도 언급되었다. 13살 우크라이나 출신 아이처럼 설정한 챗봇이 일부 심사위원을 속였고, 튜링 테스트를 통과했다는 논쟁이 생겼다. 이 사례는 지능 자체보다 평가 조건과 캐릭터 설정이 결과에 얼마나 영향을 주는지 보여준다.

3. Weak AI, Strong AI, AGI, Singularity

약한 AI와 강한 AI의 구분도 다루었다. Weak AI는 하나의 문제나 도메인을 잘 푸는 시스템에 가깝고, Strong AI는 여러 문제를 두루 다루는 범용 지능에 가깝다. GPT 계열 모델은 기존의 좁은 AI와 달리 언어, 추론, 계획, 코드, 설명을 폭넓게 다룬다는 점에서 AGI 논의와 연결된다.

구분 의미 예시적 질문
Weak AI 특정 문제나 도메인을 잘 푸는 AI 이 작업 하나를 얼마나 잘하는가?
Strong AI / AGI 여러 영역을 일반적으로 다루는 지능 새로운 문제를 다른 맥락에도 옮겨 풀 수 있는가?
Singularity 기술 성장이 통제하기 어려운 속도로 자기 개선되는 지점 AI가 스스로를 재귀적으로 개선하면 무슨 일이 생기는가?

Ray Kurzweil의 예측도 함께 언급되었다. 2029년 AGI, 2045년 특이점 같은 시간표는 확정된 미래라기보다, AI 발전 속도와 인간 사회가 어떤 상상력을 갖고 움직이는지 보여주는 프레임으로 볼 수 있다. AI와 Bio 영역이 결합되면 1년에 한 살 먹는 삶이 아니라 더 느리게 늙는 삶 같은 상상도 가능해진다.

4. Symbolism과 Connectionism

AI 기술의 큰 흐름은 Symbolism과 Connectionism으로 나누어 볼 수 있다. Connectionism은 머신러닝과 딥러닝으로 이어지는 흐름이고, Symbolism은 논리, 정리 증명, 명시적 지식 표현, 추론을 다룬다. 이 수업에서는 특히 심볼릭 AI 쪽을 중요하게 다룰 예정이다.

LLM: 언어, 패턴, 생성, 유연한 추론
+
Symbolic AI: 정확성, 규칙, 지식표현, 검증
=
Neuro-Symbolic AI: LLM과 지식그래프, 온톨로지, 추론 검증의 결합

교재로는 Artificial Intelligence: A Modern Approach가 언급되었다. Introduction, agents, search strategies, logic and inference, planning 같은 주제들이 이 과목의 골격이 된다. 중요한 점은 LLM이 갑자기 모든 것을 새로 만든 것이 아니라, 지난 75년 동안 쌓인 탐색, 추론, 계획, 지식표현의 축적 위에서 빠르게 진화하고 있다는 것이다.

5. LLM Agent와 Agentic AI

LLM 이후에는 Agentic AI가 중요해졌다. 에이전트는 목표가 주어졌을 때 환경을 지각하고, 계획하고, 행동하고, 결과를 반성하면서 목표를 달성하려는 자율 시스템이다. 단순히 답변을 생성하는 모델이 아니라, 루프를 돌며 상태를 바꾸고 도구를 사용하는 시스템에 가깝다.

구성요소 역할
Perceive 환경, 입력, 파일, 도구 결과를 관찰한다
Plan 목표를 하위 과업과 순서로 나눈다
Act 도구를 호출하거나 코드를 실행하거나 외부 시스템에 행동한다
Reflect 결과를 평가하고 다음 행동을 수정한다
Memory 단기 맥락, 장기 지식, 경험을 저장하고 불러온다

Generative Agents: Interactive Simulacra of Human Behavior도 소개되었다. LLM 기반 에이전트들이 지각하고, 메모리 스트림에서 기억을 검색하고, 계획하고, 반성하며 상호작용하는 구조를 보여준 연구다. 에이전트를 다르게 만드는 것은 LLM 성능만이 아니라 메모리, 도구, 자율성, 루프, 비용이다.

6. Planning, Memory, Tools

LLM 에이전트의 핵심 구성요소는 Planning, Memory, Tools다. Planning은 복잡한 문제를 단계별 하위 과업으로 나누고, Memory는 단기 맥락과 장기 지식, 사용자 페르소나와 경험을 관리한다. Tools는 웹 검색, 코드 실행, 파일 읽기/쓰기, API 호출 같은 외부 능력을 연결한다.

영역 핵심 개념 수업 메모
Planning ReAct, Chain of Thought, Tree of Thought 복잡한 문제를 바로 답하지 않고 절차와 분기로 구조화한다
Short-term memory In-context memory 대화창 안의 텍스트이며 토큰 한계가 있다
Long-term memory Vector DB, embedding 수많은 문서와 과거 지식을 검색 가능한 형태로 둔다
Experience memory Story-based memory 단순 문서보다 경험과 맥락을 이야기처럼 저장한다
Tools Search, code, files, API, skills, plugins, MCP LLM이 외부 세계와 연결되는 실행 능력이다

앞으로는 장기기억을 어떻게 관리할지가 중요해진다. 컨텍스트를 능동적으로 관리하는 방법으로는 압축, 요약 방향 명시, 외부 메모리 활용 등이 언급되었다. MCP는 기업이나 서비스가 열어둔 API와 도구를 에이전트가 사용할 수 있게 하는 연결 방식으로 이해할 수 있다.

7. 질문의 시대, 스토리의 시대

AI가 많은 답을 쉽게 만들어주는 시대에는 오히려 어떤 질문을 던질지가 더 중요해진다. 모든 산에 엘리베이터가 생기면 남는 질문은 "어떤 산을 오를 것인가"가 된다. 모든 정보에 쉽게 닿을 수 있다면, 진짜를 선별해주는 전문가와 조직의 전문성, 진정성, 차별화가 더 중요해진다.

Story

AI가 결과를 만들어도, 왜 이 문제를 풀어야 하는지와 어떤 맥락에서 의미가 있는지는 사람이 잡아야 한다.

Seeking

답을 받는 능력보다 좋은 질문을 찾고 검증하는 능력이 중요해진다.

Sincerity

회사의 전문성과 진정성이 있어야 AI 결과물도 신뢰 가능한 서비스로 연결된다.

Suprema Developer Portal 같은 실제 업무 맥락에 AI를 적용할 때도 마찬가지다. 이미지, 영상, 패턴, 조기 발견, 개발자 경험, 마켓플레이스, MCP와 API 연결 같은 요소를 어떻게 엮어 "손쉽게 사용할 수 있는 조건"을 만들 것인지가 중요해진다.

8. 앞으로의 역할 변화와 안전

에이전트가 강력해질수록 안전 설계도 중요해진다. LLM에는 환각, 비용, 컨텍스트 한계, 보안 문제가 있고, 에이전트는 파일, 코드, API, 비밀키에 접근할 수 있기 때문에 실수의 비용이 커진다. Claude Code 같은 도구도 비밀키와 데이터 유출을 조심해야 한다는 경고가 있었다.

사라지거나 줄어드는 일 변화하는 역할 새로 생기는 역할
반복 데이터 입력, 단순 변환, 단순 코드 번역, 정형 보고서 작성 설계, 아키텍처, 에이전트 결과 검증, PM 요구사항 명세 고도화 Agent Engineer, Prompt Engineer, AI Safety Engineer, Human-AI Interface Designer

특히 요구사항 명세 고도화가 중요하게 느껴졌다. AI가 실행을 많이 대신할수록, 사람은 무엇을 만들지, 왜 필요한지, 어떤 제약과 품질 기준이 있는지를 더 정확히 정의해야 한다. 인터페이스도 중요해진다. 앞으로는 사람이 쓰는 인터페이스뿐 아니라 AI가 사용하는 인터페이스도 설계 대상이 될 수 있다.

Keywords: artificial intelligence, intelligence, Turing Test, Chinese Room, weak AI, strong AI, AGI, singularity, symbolism, connectionism, LLM agent, ReAct, CoT, ToT, memory, tools, MCP, AI safety

Week 9 was the first class of AI Theory. The course is an intensive two-month format with three hours online and three hours offline. The first session mapped a wide territory: classical definitions of artificial intelligence, intelligence and perception, the Turing Test, the Chinese Room, weak and strong AI, AGI, symbolic AI, LLM agents, memory, tools, MCP, safety, and changing human roles.

Note. The point was not to memorize one definition of AI. The point was to see how AI imitates intelligence, how artificial problem solving is constructed, and why LLM-era agents still need classical AI concepts such as search, reasoning, planning, and symbolic verification.

1. Artificial Intelligence Is...

The class began with two starting points. Artificial intelligence can be seen as an imitation of human intelligence, and it can also be seen as artificially constructed problem solving. Intelligence combines perception, learning, knowledge, and the ability to use that knowledge.

Concept Class framing Why it matters
Artificial Intelligence Imitating human intelligence and solving problems artificially Both imitation and problem solving are necessary lenses
Intelligence Perception, learning, knowledge, and use Seeing, knowing, and using are not the same thing
Perception Receiving information from the environment Even humans miss things depending on attention

A useful point was that human perception is not complete either. People often treat perception as if it means seeing everything, but attention can make people miss what is present. To understand AI, we also have to admit that human meaning-making itself is not perfectly explained.

2. Turing Test and Chinese Room

The Turing Test comes from Alan Turing's "Computing Machinery and Intelligence." It judges intelligence through behavior and output rather than by inspecting internal structure. This looks simple, but it remained powerful for decades because it turns an abstract question into an observable test.

Turing Test

Judge whether a machine appears intelligent from conversation alone. The result depends heavily on the testing environment.

Chinese Room

If someone who does not know Chinese uses a rulebook to produce correct answers, does the system really understand Chinese?

The Chinese Room thought experiment challenges output-only evaluation. Correct answers may not mean true understanding. A counterargument is that the whole room, not only the person inside, can be treated as the machine. In the GPT era, this debate matters again: we need to decide how to challenge, verify, and interpret model outputs.

The Eugene Goostman case was also mentioned. A chatbot framed as a 13-year-old Ukrainian boy convinced some judges, leading to debate about whether it passed the Turing Test. The case shows how evaluation criteria and character framing can strongly affect the result.

3. Weak AI, Strong AI, AGI, and Singularity

Weak AI usually refers to systems that solve one task or domain well. Strong AI or AGI refers to more general intelligence across tasks and domains. GPT-like models connect to AGI discussions because they work across language, reasoning, planning, code, and explanation instead of only one narrow task.

Term Meaning Question
Weak AI AI that solves a specific task or domain How well does it do this one job?
Strong AI / AGI General intelligence across domains Can it transfer problem solving into new contexts?
Singularity A point of uncontrollable technological growth through recursive self-improvement What happens if AI improves itself?

Ray Kurzweil's timeline was introduced as part of this discussion, including forecasts such as AGI in 2029 and singularity in 2045. These are not guaranteed dates, but they are useful frames for understanding how people imagine the speed of AI and bio-technology development.

4. Symbolism and Connectionism

AI technology can be viewed through two large traditions. Connectionism leads toward machine learning and deep learning. Symbolism focuses on logic, theorem proving, explicit knowledge representation, and reasoning. This course will emphasize the symbolic side.

LLM: language, patterns, generation, flexible reasoning

5. LLM Agents and Agentic AI

After LLMs, Agentic AI became a major topic. An agent is an autonomous system that receives a goal, perceives its environment, plans, acts, reflects on results, and tries to achieve the goal. It is closer to a looped system that changes state and uses tools than to a model that only produces answers.

Component Role
Perceive Observe inputs, environment, files, and tool results
Plan Break goals into subtasks and sequence
Act Call tools, run code, or interact with external systems
Reflect Evaluate outcomes and revise the next action
Memory Store and retrieve short-term context, long-term knowledge, and experience

Generative Agents: Interactive Simulacra of Human Behavior was introduced as an example of LLM-based agents that perceive, retrieve memories from a memory stream, plan, reflect, and interact. What differentiates agents is not only the LLM but memory, tools, autonomy, loops, and cost.

6. Planning, Memory, Tools

The core components of LLM agents are planning, memory, and tools. Planning structures complex problems into subtasks. Memory manages short-term context, long-term knowledge, user persona, and experience. Tools connect the LLM to web search, code execution, file read/write, API calls, skills, plugins, and MCP.

Area Key idea Class note
Planning ReAct, Chain of Thought, Tree of Thought Problems should be structured into procedures and branches
Short-term memory In-context memory Text inside the conversation window, limited by tokens
Long-term memory Vector DB, embeddings Large document stores made searchable
Experience memory Story-based memory Experience and context stored as stories rather than isolated facts
Tools Search, code, files, APIs, skills, plugins, MCP Execution abilities that connect LLMs to the outside world

Managing long-term memory will become increasingly important. Active context management includes compaction, directed summarization, and external memory. MCP can be understood as a way for agents to use APIs and tools exposed by companies and services.

7. The Age of Questions and Stories

When AI can generate many answers easily, the important question becomes what to ask. If every mountain has an elevator, the remaining question is which mountain to climb. When information becomes easy to reach, experts who can select what is real and meaningful become more important.

Story

AI can produce outputs, but humans still need to define why the problem matters and what context gives it meaning.

Seeking

The ability to find and verify good questions becomes more important than simply receiving answers.

Sincerity

Company expertise and authenticity are needed before AI outputs can become trustworthy services.

The same applies when thinking about applying AI to a developer portal such as Suprema's. Images, video, pattern detection, early discovery, developer experience, marketplaces, MCP, and API connections all matter when building conditions that make AI genuinely usable.

8. Role Changes and Safety

As agents become stronger, safety design becomes more important. LLMs have hallucinations, cost, context limits, and security risks. Agents can access files, code, APIs, and secret keys, which raises the cost of mistakes. The class also warned about secret key and data leakage risks in tools such as Claude Code.

Work that shrinks Roles that change New roles
Repeated data entry, simple transformation, simple code translation, standardized reports Design, architecture, agent output verification, stronger PM requirement specification Agent Engineer, Prompt Engineer, AI Safety Engineer, Human-AI Interface Designer

The phrase that stood out most was stronger requirement specification. As AI handles more execution, humans must define what to build, why it matters, and what constraints and quality standards apply. Interfaces also become important, including interfaces used not only by humans but by AI systems.
