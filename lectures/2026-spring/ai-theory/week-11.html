Local Search

Keywords: local search, 100-bit string optimization, hill climbing, random restart, random walk, graph search, simulated annealing, temperature, acceptance probability, neighborhood, objective value

11주차는 전역 탐색 문제를 모두 펼쳐 보는 방식에서 벗어나, 현재 해 주변을 움직이며 더 좋은 해를 찾아가는 지역 탐색(local search)을 실습한 시간이었다. 과제는 100-bit string 최적화였고, 수업에서는 hill climbing을 random-restart 방식으로 진행하고, 그래프 위에서의 random walk를 살펴본 뒤, simulated annealing의 용어와 알고리즘에 익숙해지는 것을 목표로 했다.

메모. 지역 탐색은 전체 경로보다 현재 해의 품질과 이웃 해의 품질을 비교하는 데 집중한다. 그래서 구현은 단순해 보이지만, 지역 최적해에 갇히지 않기 위한 restart, random move, 확률적 수용 규칙이 중요해진다.

1. 과제: 100-bit string 최적화

100-bit string 최적화 문제에서는 하나의 해를 길이 100의 0/1 리스트로 표현한다. 각 위치의 비트를 바꾸면 다른 후보 해가 되고, 평가 함수는 그 후보 해가 얼마나 좋은지를 점수로 반환한다. Local search에서는 전체 가능한 문자열을 모두 나열하지 않고, 현재 문자열에서 한 비트 또는 일부 비트를 바꾼 이웃 해를 비교하며 점수를 개선한다.

항목 100-bit string에서의 의미
State 길이 100의 0/1 문자열 또는 리스트
Neighbor 현재 문자열에서 한 비트 등을 뒤집어 만든 후보
Objective value 후보 문자열을 평가한 점수
Search goal 제한된 반복 안에서 더 높은 점수의 문자열 찾기

이 과제에서 중요한 점은 알고리즘이 "항상 좋아지는 이동만 할 것인가", "나쁜 이동도 가끔 받아들일 것인가", "한 번의 시작점에 의존할 것인가"를 직접 비교하는 것이다. 같은 100-bit 표현을 쓰더라도 neighbor 생성 방식, restart 횟수, temperature 감소 방식에 따라 결과가 달라진다.

state = random_100_bit_string()
neighbor = flip_one_bit(state)
delta = value(neighbor) - value(state)

2. Hill Climbing과 Random-Restart

Hill climbing은 현재 해의 이웃 중 더 좋은 해를 선택해 조금씩 올라가는 방식이다. 목표가 값을 최대화하는 문제라면, 현재 해보다 높은 값을 가진 이웃을 찾고 그쪽으로 이동한다. 더 좋은 이웃이 없으면 현재 위치는 지역 최적해가 된다.

구성 요소 의미
Current solution 현재 평가 중인 해
Neighbor 현재 해를 조금 바꿔 만든 후보 해
Value / score 해의 품질을 평가하는 목적 함수 값
Local optimum 주변 이웃보다 나쁘지 않지만 전체 최적은 아닐 수 있는 해

문제는 hill climbing이 한 번의 시작점에 크게 의존한다는 점이다. 그래서 이번 수업에서는 random-restart hill climbing으로 진행했다. 무작위 초기 해에서 hill climbing을 여러 번 실행하고, 각 실행에서 얻은 결과 중 가장 좋은 해를 최종 결과로 선택한다.

random initial solution -> hill climbing -> local optimum
repeat several times
choose the best result among restarts

3. Random Walk on a Graph

Random walk는 그래프에서 현재 노드의 이웃 중 하나를 무작위로 선택해 이동하는 방식이다. BFS나 A*처럼 명확한 목표 방향이나 우선순위를 두기보다, 움직임 자체에 확률성을 둔다. 따라서 좋은 해를 보장하기보다는 그래프 구조를 탐색하거나, 결정적 탐색이 놓치는 경로를 우연히 방문하는 데 의미가 있다.

비교 Hill climbing Random walk
이동 기준 더 좋은 이웃을 우선 선택 이웃 중 하나를 무작위 선택
방향성 목적 함수 값의 개선 방향 확률적 이동
장점 좋은 이웃이 있으면 빠르게 개선 특정 방향에 갇히지 않고 이동 가능
주의점 지역 최적해에 갇힐 수 있음 무작위 이동만으로는 비효율적일 수 있음

4. Simulated Annealing 용어

Simulated annealing은 hill climbing처럼 이웃 해로 이동하지만, 항상 좋아지는 방향만 선택하지 않는다. 새로운 해가 더 나쁘더라도 일정 확률로 받아들인다. 초반에는 temperature가 높아 나쁜 해도 비교적 자주 받아들이고, 시간이 지나 temperature가 낮아지면 더 보수적으로 움직인다.

용어 설명
Temperature T 나쁜 해를 받아들일 가능성을 조절하는 값
Cooling schedule 반복이 진행될수록 T를 낮추는 규칙
Delta E value(neighbor) - value(current). 최대화 문제에서는 양수면 더 좋은 해다.
Acceptance probability 나쁜 해를 확률적으로 받아들이는 기준
Delta E = value(neighbor) - value(current)
if Delta E >= 0: accept neighbor
if Delta E < 0: accept neighbor when e^(Delta E / T) > rand(0, 1)

여기서 최대화 문제를 기준으로 하면, 이웃 해가 더 좋을 때는 바로 교체한다. 이웃 해가 더 나쁘면 e^(Delta E / T)와 rand(0, 1)을 비교해 확률적으로 교체한다. Delta E가 음수여도 T가 충분히 크면 받아들일 가능성이 남고, T가 작아지면 받아들이기 어려워진다.

5. SA Algorithm 흐름

Simulated annealing은 지역 최적해를 피하기 위해 일부러 손해 보는 이동을 허용한다. 하지만 무작정 흔들리는 것이 아니라, temperature를 점점 낮추며 초반에는 탐색을 넓게 하고 후반에는 좋은 해 주변에 수렴하게 만든다.

current = random_solution()
best = current
T = initial_temperature

while T > minimum_temperature:
    neighbor = random_neighbor(current)
    delta = value(neighbor) - value(current)

    if delta >= 0:
        current = neighbor
    elif exp(delta / T) > rng.random():
        current = neighbor

    if value(current) > value(best):
        best = current

    T = cool(T)

return best

  • 과제는 100-bit string을 local search로 최적화하는 문제였다.
  • Hill climbing은 더 좋은 이웃으로 이동하지만, 지역 최적해에 갇힐 수 있다.
  • Random-restart는 시작점을 여러 번 바꾸어 hill climbing의 약점을 줄인다.
  • Random walk는 그래프에서 이웃을 무작위로 선택해 이동하는 확률적 탐색이다.
  • Simulated annealing은 더 나쁜 해도 확률적으로 받아들이며 탐색 범위를 넓힌다.
  • SA의 핵심 조건은 Delta E = value(neighbor) - value(current)와 e^(Delta E / T) > rand(0, 1)이다.

Keywords: local search, 100-bit string optimization, hill climbing, random restart, random walk, graph search, simulated annealing, temperature, acceptance probability, neighborhood, objective value

Week 11 practiced local search: moving around the neighborhood of a current solution instead of expanding the entire search space. The assignment was 100-bit string optimization, and the class used random-restart hill climbing, random walk on a graph, and simulated annealing to become familiar with the core terms and algorithmic behavior.

Note. Local search focuses on the value of the current solution and its neighbors rather than full path construction. The implementation can look simple, but restart, random movement, and probabilistic acceptance matter because a deterministic climb can get stuck.

1. Assignment: 100-bit String Optimization

In the 100-bit string optimization problem, one solution is represented as a length-100 list of 0s and 1s. Flipping bits creates candidate solutions, and the evaluation function returns how good each candidate is. Local search does not enumerate every possible string; it compares neighboring strings near the current one and tries to improve the score.

Item Meaning in a 100-bit string problem
State A length-100 binary string or list
Neighbor A candidate made by flipping one or more bits
Objective value The score returned by evaluating the candidate string
Search goal Find a higher-scoring string within a limited number of iterations

The assignment made the tradeoffs visible: whether the algorithm accepts only improving moves, occasionally accepts worse moves, or depends too much on one starting point. With the same 100-bit representation, results can change depending on neighbor generation, restart count, and the temperature schedule.

state = random_100_bit_string()
neighbor = flip_one_bit(state)
delta = value(neighbor) - value(state)

2. Hill Climbing and Random-Restart

Hill climbing repeatedly moves from the current solution to a better neighboring solution. In a maximization problem, it looks for a neighbor with a higher value and moves there. If no better neighbor exists, the current state is a local optimum.

Component Meaning
Current solution The solution currently being evaluated
Neighbor A candidate solution made by slightly changing the current solution
Value / score The objective-function value that evaluates solution quality
Local optimum A solution that is not worse than its neighbors but may not be globally optimal

The weakness is dependence on the starting point. The class therefore used random-restart hill climbing: run hill climbing from several random initial solutions, then keep the best result across all restarts.

random initial solution -> hill climbing -> local optimum
repeat several times
choose the best result among restarts

3. Random Walk on a Graph

A random walk moves through a graph by randomly choosing one of the current node's neighbors. Unlike BFS or A*, it does not use a clear priority rule or goal direction. Its value is in probabilistic movement: it may visit paths that a deterministic rule would not choose.

Comparison Hill climbing Random walk
Move rule Prefer a better neighbor Randomly choose a neighbor
Direction Objective-value improvement Probabilistic movement
Strength Improves quickly when a better neighbor exists Can keep moving without committing to one local direction
Risk Can get stuck in a local optimum Pure randomness can be inefficient

4. Simulated Annealing Terms

Simulated annealing also moves to neighboring solutions, but it does not accept only improving moves. If a new solution is worse, it may still replace the current solution with some probability. Early in the run, the temperature is high and worse moves are accepted more often. Later, the temperature decreases and the search becomes more conservative.

Term Explanation
Temperature T Controls how likely the algorithm is to accept worse solutions
Cooling schedule The rule that lowers T as iterations continue
Delta E value(neighbor) - value(current). In maximization, a positive value is better.
Acceptance probability The probabilistic rule used to accept a worse solution
Delta E = value(neighbor) - value(current)
if Delta E >= 0: accept neighbor
if Delta E < 0: accept neighbor when e^(Delta E / T) > rand(0, 1)

For a maximization problem, a better neighbor is accepted immediately. If the neighbor is worse, the algorithm compares e^(Delta E / T) with rand(0, 1). Even when Delta E is negative, a high T leaves a meaningful chance of acceptance; as T gets smaller, acceptance becomes harder.

5. SA Algorithm Flow

Simulated annealing avoids local optima by allowing occasional worse moves. It is not random wandering forever: the temperature gradually cools, so the early stage explores broadly and the later stage settles around stronger solutions.

current = random_solution()
best = current
T = initial_temperature

while T > minimum_temperature:
    neighbor = random_neighbor(current)
    delta = value(neighbor) - value(current)

    if delta >= 0:
        current = neighbor
    elif exp(delta / T) > rng.random():
        current = neighbor

    if value(current) > value(best):
        best = current

    T = cool(T)

return best
