Genetic Algorithm Practice

Keywords: Zombie Apocalypse, evaluate-search.py, genetic algorithm, selection, tournament selection, roulette wheel selection, one-point crossover, mutation, repair, knapsack constraint

12주차는 좀비 아포칼립스 상황을 배경으로 한 evaluate-search.py 관련 문제를 풀며, 유전 알고리즘의 기본 연산자를 직접 구현하고 비교한 실습 시간이었다. 핵심은 population에서 부모를 선택하고, crossover로 자식을 만들고, mutate로 변화를 주며, capacity 제약을 넘는 해를 repair하는 흐름을 코드로 확인하는 것이었다.

메모. 아래 코드는 수업 중 나온 학생 구현 예시를 이름과 학번 없이 정리한 것이다. 같은 연산자라도 선택 방식, 복사 여부, 가중치 처리, 제약 조건 복구 방식에 따라 탐색 결과가 달라질 수 있다.

1. 전체 실습 흐름

문제는 제한된 용량 안에서 생존에 도움이 되는 아이템 조합을 고르는 형태로 볼 수 있다. 각 individual은 0과 1로 이루어진 리스트이고, 각 비트는 해당 아이템을 선택했는지 나타낸다. score는 선택한 아이템의 가치 또는 적합도를 나타내며, 알고리즘은 더 좋은 조합을 만들기 위해 세대를 반복한다.

연산자 역할
Select population에서 부모 후보를 고른다
Crossover 두 부모의 일부를 섞어 child를 만든다
Mutate child의 일부 비트를 낮은 확률로 뒤집는다
Repair capacity를 초과한 해를 제약 조건 안으로 되돌린다

2. evaluate-search.py에서 확인할 흐름

evaluate-search.py 실습에서는 각 연산자를 따로 보는 데서 끝나지 않고, 전체 탐색 루프 안에서 어떤 역할을 하는지 확인했다. 좋은 select는 부모 후보를 안정적으로 고르고, crossover와 mutate는 새로운 후보를 만들며, repair는 문제의 제약 조건을 다시 만족시킨다.

population = initialize_population()

for generation in range(max_generations):
    scores = evaluate(population)
    next_population = []

    while len(next_population) < len(population):
        parent_a = select(population, scores, rng)
        parent_b = select(population, scores, rng)
        child = crossover(parent_a, parent_b, rng)
        child = mutate(child, rng)
        child = repair(child, rng)
        next_population.append(child)

    population = next_population

여기서 핵심은 함수 시그니처를 맞추고, 무작위 선택에는 전달받은 rng를 사용하는 것이다. 그래야 같은 seed에서 결과를 재현할 수 있고, 서로 다른 구현을 공정하게 비교할 수 있다.

3. Select: Tournament 방식

Tournament selection은 population에서 몇 개의 후보를 무작위로 뽑고, 그중 score가 가장 높은 individual을 부모로 선택한다. 아래 예시는 세 개의 후보를 뽑는 tournament 방식이다.

def select(population: list[list[int]], scores: list[int], rng: random.Random) -> list[int]:
    idx1 = rng.randint(0, len(population) - 1)
    idx2 = rng.randint(0, len(population) - 1)
    idx3 = rng.randint(0, len(population) - 1)
    best_idx = idx1
    if scores[idx2] > scores[best_idx]:
        best_idx = idx2
    if scores[idx3] > scores[best_idx]:
        best_idx = idx3
    return population[best_idx].copy()

이 방식은 높은 점수를 가진 individual이 선택될 가능성을 높이지만, 완전히 최고 점수만 고르는 방식은 아니다. 후보를 무작위로 뽑기 때문에 population의 다양성이 어느 정도 유지된다.

4. Select: Roulette Wheel 방식

Roulette wheel selection은 score를 weight처럼 사용해 부모를 뽑는다. 점수가 높을수록 선택될 확률이 커진다.

def select(population: list[list[int]], scores: list[int], rng: random.Random) -> list[int]:
    parent_cand = rng.choices(range(len(population)), weights=scores, k=1)
    parent = population[parent_cand[0]]
    return parent

이 방식은 점수 차이를 확률에 직접 반영한다. 다만 score가 모두 0이거나 음수가 섞이는 경우에는 별도 처리가 필요할 수 있다.

5. Crossover: One-Point Crossover

One-point crossover는 한 지점을 기준으로 두 부모를 잘라 앞부분은 첫 번째 부모에서, 뒷부분은 두 번째 부모에서 가져와 child를 만든다.

def crossover(parent_a: list[int], parent_b: list[int], rng: random.Random) -> list[int]:
    n = len(parent_a)
    point = rng.randint(1, n - 1)
    child = parent_a[:point] + parent_b[point:]
    return child

교차 지점이 어디로 잡히는지에 따라 부모의 특성이 다르게 섞인다. 이 연산자는 단순하지만, 두 부모의 조합을 새롭게 만들어 population을 탐색하게 해준다.

6. Mutate: Bit Flip

Mutation은 자식 individual의 일부 비트를 낮은 확률로 뒤집는다. 아래 예시는 비트 길이가 n일 때 각 위치를 1 / n 확률로 바꾼다.

def mutate(individual: list[int], rng: random.Random) -> list[int]:
    child = individual[:]
    pm = 1 / len(child)
    for i in range(len(child)):
        if rng.random() < pm:
            child[i] = 1 - child[i]
    return child

Mutation은 너무 크면 좋은 구조를 쉽게 깨고, 너무 작으면 새로운 조합을 찾기 어렵다. 1 / len(child)는 한 individual에서 평균적으로 한 비트 정도가 바뀌도록 하는 간단한 기준이다.

7. Repair: Capacity 제약 복구

Zombie Apocalypse 문제처럼 선택할 수 있는 총 무게가 제한된 문제에서는 crossover와 mutation 후 capacity를 넘는 해가 생길 수 있다. Repair는 이런 해를 다시 유효한 해로 되돌리는 과정이다.

def repair(individual: list[int], rng: random.Random) -> list[int]:
    total_weight = sum(ITEMS[i]["weight"] for i, bit in enumerate(individual) if bit)

    if total_weight > CAPACITY:
        efficiency = [
            (i, ITEMS[i]["value"] / ITEMS[i]["weight"])
            for i in range(len(individual))
            if individual[i] == 1
        ]
        efficiency.sort(key=lambda x: x[1])

        while total_weight > CAPACITY and efficiency:
            idx, _ = efficiency.pop(0)
            individual[idx] = 0
            total_weight -= ITEMS[idx]["weight"]

    return individual

위 방식은 선택된 아이템 중 value/weight 비율이 낮은 항목부터 제거한다. 즉, 무게 제한을 맞추되 상대적으로 효율이 낮은 선택을 먼저 포기하는 repair 전략이다.

Week 12 used a Zombie Apocalypse scenario and an evaluate-search.py practice problem to implement and compare basic genetic-algorithm operators. The core loop was to select parents from a population, create children through crossover, introduce variation through mutation, and repair solutions that violate the capacity constraint.

Note. The examples below reorganize in-class student implementations without names or student IDs. Even for the same operator, selection rules, copying behavior, weight handling, and repair strategy can change the search result.

1. Practice Flow

The problem can be viewed as selecting useful survival items under a capacity limit. Each individual is a list of 0s and 1s, and each bit indicates whether an item is selected. The score represents value or fitness, and the algorithm repeats generations to find stronger combinations.

Operator Role
Select Choose parent candidates from the population
Crossover Combine parts of two parents to create a child
Mutate Flip some bits of the child with low probability
Repair Bring an overweight solution back within the constraint

2. Flow Checked in evaluate-search.py

The evaluate-search.py practice did not treat each operator as an isolated snippet. The point was to see how each function works inside the full search loop. Selection chooses parents, crossover and mutation create new candidates, and repair restores feasibility under the problem constraint.

population = initialize_population()

for generation in range(max_generations):
    scores = evaluate(population)
    next_population = []

    while len(next_population) < len(population):
        parent_a = select(population, scores, rng)
        parent_b = select(population, scores, rng)
        child = crossover(parent_a, parent_b, rng)
        child = mutate(child, rng)
        child = repair(child, rng)
        next_population.append(child)

    population = next_population

The key implementation requirement is to keep the expected function signatures and use the provided rng for randomness. That makes results reproducible under the same seed and allows different implementations to be compared fairly.

3. Select: Tournament Selection

Tournament selection samples several candidates from the population and chooses the one with the highest score. This example uses a tournament of three candidates.

def select(population: list[list[int]], scores: list[int], rng: random.Random) -> list[int]:
    idx1 = rng.randint(0, len(population) - 1)
    idx2 = rng.randint(0, len(population) - 1)
    idx3 = rng.randint(0, len(population) - 1)
    best_idx = idx1
    if scores[idx2] > scores[best_idx]:
        best_idx = idx2
    if scores[idx3] > scores[best_idx]:
        best_idx = idx3
    return population[best_idx].copy()

This raises the chance of selecting a stronger individual without always selecting the global best. Because candidates are sampled randomly, some population diversity remains.

4. Select: Roulette Wheel Selection

Roulette wheel selection uses scores as weights. Higher-score individuals are more likely to be selected.

def select(population: list[list[int]], scores: list[int], rng: random.Random) -> list[int]:
    parent_cand = rng.choices(range(len(population)), weights=scores, k=1)
    parent = population[parent_cand[0]]
    return parent

This directly maps score differences into probability. If all scores are zero or if negative scores appear, the implementation may need extra handling.

5. Crossover: One-Point Crossover

One-point crossover chooses one split point. The child takes the front part from the first parent and the back part from the second parent.

def crossover(parent_a: list[int], parent_b: list[int], rng: random.Random) -> list[int]:
    n = len(parent_a)
    point = rng.randint(1, n - 1)
    child = parent_a[:point] + parent_b[point:]
    return child

The split point determines how the parents' traits are mixed. The operator is simple, but it lets the search explore new combinations of existing patterns.

6. Mutate: Bit Flip

Mutation flips some bits of the child with low probability. In this example, each position changes with probability 1 / n, where n is the bit length.

def mutate(individual: list[int], rng: random.Random) -> list[int]:
    child = individual[:]
    pm = 1 / len(child)
    for i in range(len(child)):
        if rng.random() < pm:
            child[i] = 1 - child[i]
    return child

If mutation is too strong, it breaks useful structures too easily. If it is too weak, the algorithm may not discover enough new combinations. 1 / len(child) is a simple rule that changes about one bit on average.

7. Repair: Capacity Constraint

In a problem with a strict capacity limit, crossover and mutation can produce overweight solutions. Repair restores such individuals to a valid state.

def repair(individual: list[int], rng: random.Random) -> list[int]:
    total_weight = sum(ITEMS[i]["weight"] for i, bit in enumerate(individual) if bit)

    if total_weight > CAPACITY:
        efficiency = [
            (i, ITEMS[i]["value"] / ITEMS[i]["weight"])
            for i in range(len(individual))
            if individual[i] == 1
        ]
        efficiency.sort(key=lambda x: x[1])

        while total_weight > CAPACITY and efficiency:
            idx, _ = efficiency.pop(0)
            individual[idx] = 0
            total_weight -= ITEMS[idx]["weight"]

    return individual

This strategy removes selected items with the lowest value-to-weight ratio first. It tries to satisfy the capacity limit while giving up relatively inefficient choices first.
