Invertibility · LU · Blocks

정리 기준: 04. 역행렬 · 기본행렬, 활용, LU 분해, 블록 역행렬을 중심으로 재구성

역행렬은 변환을 되돌리는 행렬이지만 모든 정방행렬에 존재하지 않는다. 이번 주의 핵심은 공식을 외우는 것이 아니라 행 연산·해의 유일성·분해 구조가 같은 가역성 개념을 서로 다르게 보여 준다는 점이다.

1. 기본행렬은 하나의 행 연산을 행렬 곱으로 표현한다

  • 단위행렬에 행 교환, 0이 아닌 상수배, 다른 행의 배수 더하기를 적용하면 해당 기본행렬을 얻는다.
  • 기본행렬을 왼쪽에서 곱하면 원래 행렬에 같은 행 연산이 적용되며 각 기본행렬 자체도 가역이다.
  • A를 I로 바꾸는 행 연산이 존재하면 같은 연산을 I에 적용한 결과가 A⁻¹이 된다.

2. 목적에 따라 역행렬·소거·분해를 선택한다

상황적합한 관점이유
역행렬 자체 필요[A | I] 행 소거가역성 판정과 계산을 동시에 수행
Ax = b 한 번 풀이직접 소거불필요한 전체 역행렬을 피함
같은 A, 여러 bLU 분해분해를 재사용해 전진·후진대입
부분 구조가 큰 행렬블록 연산작은 하위 문제로 나눔

3. LU 분해는 소거 과정을 재사용 가능한 형태로 저장한다

A = LU에서 L은 소거 배수를 담는 하삼각행렬, U는 소거 뒤의 상삼각행렬이다. Ax = b를 Ly = b와 Ux = y로 나누면 전진대입과 후진대입으로 해결할 수 있다. 블록행렬도 조건을 만족하면 부분행렬의 역과 결합 구조를 이용해 다룬다.

[A | I] ~ [I | A⁻¹] · A = LU · Ax = b ⇒ Ly = b, Ux = y

계산 원칙. x = A⁻¹b는 이론적으로 명확하지만 수치 계산에서는 역행렬을 명시적으로 만드는 것보다 연립방정식을 직접 푸는 편이 보통 효율적이고 안정적이다. 목적과 반복 횟수에 맞는 절차를 고른다.

핵심 정리

  • 가역행렬은 양쪽에서 곱해 단위행렬을 만드는 유일한 역행렬을 가지며 행 연산으로 I까지 변환할 수 있다.
  • 동차 시스템 Ax = 0의 해가 영벡터뿐이라는 사실은 정방행렬의 가역성과 연결된다.
  • LU 분해와 블록 구조는 큰 문제의 계산을 작은 삼각 또는 부분 문제로 분해한다.

실전 팁

  • 역행렬을 구할 때 [A | I] 양쪽에 정확히 같은 행 연산을 적용하고 왼쪽이 I인지 먼저 확인한다.
  • 계산한 후보 B에 대해 AB = I와 BA = I를 검산하고 한쪽 결과만으로 성급히 결론내리지 않는다.
  • 같은 계수행렬에 여러 우변이 주어지면 A의 LU 분해를 한 번만 수행하고 각 우변에 대입 과정을 반복한다.

Basis: 04. Inverse Matrices · Reorganized around elementary matrices, applications, LU decomposition, and block inverses

An inverse matrix reverses a transformation, but not every square matrix has one. The central idea is not a memorized formula: row operations, uniqueness of solutions, and factorization reveal the same notion of invertibility from different directions.

1. An Elementary Matrix Represents One Row Operation

  • Apply a row swap, nonzero scaling, or row replacement to the identity matrix to obtain the corresponding elementary matrix.
  • Left multiplication by that elementary matrix applies the same row operation to another matrix, and every elementary matrix is itself invertible.
  • If a sequence of row operations changes A into I, applying the same sequence to I produces A⁻¹.

2. Choose Inversion, Elimination, or Factorization for the Goal

SituationUseful viewReason
The inverse itself is requiredReduce [A | I]Tests and computes invertibility together
Solve Ax = b onceDirect eliminationAvoids constructing the full inverse
Same A, many b vectorsLU decompositionReuses factors with forward/back substitution
Large matrix with subsystemsBlock operationsSplits the task into smaller components

3. LU Decomposition Stores Elimination for Reuse

In A = LU, L is lower triangular and stores elimination multipliers, while U is the upper-triangular result of elimination. The equation Ax = b becomes Ly = b followed by Ux = y, solved by forward and backward substitution. Under suitable conditions, a block matrix can similarly be handled through its submatrices and coupling structure.

[A | I] ~ [I | A⁻¹] · A = LU · Ax = b ⇒ Ly = b, Ux = y

Computational principle. Although x = A⁻¹b is theoretically clear, numerical work usually solves the system directly instead of explicitly forming an inverse. Select a procedure according to the goal and how often the coefficient matrix will be reused.

Key Takeaways

  • An invertible matrix has a unique two-sided inverse and can be reduced to the identity through elementary row operations.
  • For a square matrix, the homogeneous system Ax = 0 having only the zero solution is another expression of invertibility.
  • LU and block structures reduce a large computation to smaller triangular or subsystem problems.

Practical Tips

  • When reducing [A | I], apply every row operation to both halves and verify that the left block is exactly I before reading the inverse.
  • Check a candidate B with both AB = I and BA = I rather than trusting one partial computation.
  • When many right-hand sides share the same coefficient matrix, factor A once and repeat only the substitution steps for each b.
