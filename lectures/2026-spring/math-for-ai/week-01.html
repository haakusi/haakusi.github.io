정리 기준: 01. 선형대수학의 개요 · 기본 개념, 범위, 응용을 중심으로 재구성

선형대수는 숫자 계산만을 위한 과목이 아니라 데이터를 벡터와 행렬로 표현하고, 관계를 함수와 변환으로 설명하는 언어다. 첫 주에는 이후 모든 장에서 반복되는 수학적 대상과 표기 사이의 연결을 세운다.

1. 집합과 사상으로 대상과 관계를 정의한다

  • 집합은 조건을 만족하는 대상의 모임이며 합집합·교집합·차집합은 데이터 범주를 결합하거나 비교하는 기본 연산이다.
  • 사상 또는 함수는 정의역의 각 원소를 공역의 한 원소에 대응시킨다. 실제로 도달한 값의 집합이 치역이다.
  • 단사는 서로 다른 입력을 서로 다른 출력에, 전사는 공역의 모든 원소에, 전단사는 두 조건을 모두 만족시켜 역함수를 가능하게 한다.

3. 데이터는 형태에 맞는 배열로 바뀐다

표는 행과 열의 행렬, 회색조 영상은 픽셀값 행렬, 텍스트는 단어나 토큰의 벡터, 그래프는 인접행렬이나 특징행렬로 표현할 수 있다. 표현이 정해지면 행렬 곱으로 변환을 적용하고 벡터 사이의 관계를 계산할 수 있다.

이미지 H × W → 픽셀 행렬 · 문서 → 토큰/특징 벡터 · 그래프 → 인접행렬 A · 변환 → y = Ax

연결 관점. 벡터는 하나의 데이터 항목, 행렬은 여러 벡터의 묶음이자 변환, 함수는 입력과 출력의 규칙이다. 같은 표기를 계산 도구와 의미 표현이라는 두 관점에서 함께 읽는다.

핵심 정리

  • 집합은 대상을, 함수는 대응 관계를, 벡터와 행렬은 수치 표현을 제공한다.
  • 정의역·공역·치역과 단사·전사·전단사의 차이는 역변환 가능성을 이해하는 기반이다.
  • AI 데이터의 종류가 달라도 적절한 벡터와 행렬로 바꾸면 공통된 선형대수 연산을 적용할 수 있다.

실전 팁

  • 새 표기를 볼 때 대상의 종류, 크기, 입력 공간, 출력 공간을 먼저 옆에 적는다.
  • 표·영상·문장·그래프 각각을 벡터 또는 행렬로 바꾸는 작은 예를 만들어 행과 열의 의미를 말해 본다.
  • 함수 예시마다 정의역·공역·치역을 그린 뒤 단사·전사·전단사 중 어디에 해당하는지 확인한다.

Basis: 01. Linear Algebra Overview · Reorganized around foundational objects, scope, and applications

Linear algebra is not merely a collection of numerical procedures. It is a language for representing data as vectors and matrices and describing relationships as functions and transformations. This week builds the object-to-notation connections that recur throughout the course.

1. Sets and Mappings Define Objects and Relations

  • A set collects objects satisfying a condition; union, intersection, and difference are basic ways to combine or compare data categories.
  • A mapping or function sends every element of a domain to one element of a codomain. The values actually reached form the range.
  • An injective map keeps distinct inputs distinct, a surjective map reaches the entire codomain, and a bijective map satisfies both conditions and can support an inverse.

2. Separate the Four Axes of Linear Algebra

3. Data Becomes an Array Chosen for Its Structure

A table becomes a row-column matrix, a grayscale image a matrix of pixel intensities, text a vector of token or feature values, and a graph an adjacency or feature matrix. Once the representation is explicit, matrix multiplication applies a transformation and vector operations compare or combine observations.

Image H × W → pixel matrix · document → token/feature vector · graph → adjacency matrix A · transformation → y = Ax

Connection view. A vector can represent one observation; a matrix can be a collection of vectors or an operator; a function supplies the input-output rule. Read notation both as a computational object and as a statement about meaning.

Key Takeaways

  • Sets describe objects, functions describe correspondences, and vectors and matrices provide numerical representations.
  • Domain, codomain, range, injectivity, surjectivity, and bijectivity provide the foundation for reasoning about invertible mappings.
  • Different AI data types can share the same linear-algebra operations after an appropriate vector or matrix representation is chosen.

Practical Tips

  • For every new symbol, annotate its object type, dimensions, input space, and output space before calculating.
  • Create one small table, image, sentence, and graph example; encode each as a vector or matrix and explain every row and column.
  • Draw the domain, codomain, and range of sample functions, then classify each map as injective, surjective, bijective, or neither.
