정리 기준: 03. 행렬 · 기본 연산, 역행렬, 특별한 행렬을 중심으로 재구성

행렬은 수를 직사각 배열로 모은 객체이면서 벡터를 다른 벡터로 바꾸는 연산자다. AI에서는 배치 데이터와 가중치를 같은 표기로 다루므로 크기 규칙과 곱셈 순서가 곧 모델의 데이터 흐름이 된다.

1. 크기를 먼저 읽으면 가능한 연산이 보인다

  • m × n 행렬에는 m개의 행과 n개의 열이 있고 각 열은 m차원 열벡터로 볼 수 있다.
  • 덧셈과 뺄셈은 크기가 같은 행렬끼리 성분별로 수행하고 스칼라배는 모든 성분에 같은 수를 곱한다.
  • AB가 가능하려면 A의 열 수와 B의 행 수가 같아야 하며 결과 크기는 A의 행 수 × B의 열 수다.

2. 구조가 있는 행렬은 계산을 단순화한다

행렬정의의미
단위행렬 I주대각선은 1, 나머지는 0곱셈에서 값을 보존
대각·삼각행렬특정 위치 밖의 성분이 0저장과 계산을 단순화
대칭행렬Aᵀ = A쌍 관계가 방향에 무관
블록행렬작은 행렬을 성분처럼 배치큰 시스템의 부분 구조 표현

3. 행렬 곱과 전치는 순서에 민감하다

AB의 각 성분은 A의 행과 B의 열의 내적으로 만들어진다. 일반적으로 AB ≠ BA이므로 연산의 적용 순서를 바꾸면 결과가 달라진다. 전치는 행과 열을 뒤집고 곱 전체를 전치할 때 내부 순서도 반대로 바꾼다.

A(m × n)B(n × p) → AB(m × p) · (AB)ᵀ = BᵀAᵀ · AA⁻¹ = A⁻¹A = I

차원 감각. 신경망에서 데이터 X, 가중치 W, 출력 Y의 크기를 적으면 XW가 가능한지와 출력 특징 수를 계산 전에 알 수 있다. 크기 불일치는 단순 오류가 아니라 잘못 설계된 변환을 뜻한다.

핵심 정리

  • 행렬 연산은 성분 규칙뿐 아니라 입력·출력 차원을 보존하는 형태 규칙을 따른다.
  • 행렬 곱은 일반적으로 교환되지 않지만 결합법칙을 만족하므로 괄호를 계산 효율에 맞게 선택할 수 있다.
  • 전치·대칭·대각·삼각·블록 구조는 행렬의 의미와 효율적인 계산법을 함께 알려 준다.

실전 팁

  • 모든 곱셈 위에 행렬 크기를 써서 안쪽 차원 일치와 바깥쪽 결과 크기를 먼저 확인한다.
  • AB와 BA를 작은 수치 예로 직접 계산해 순서가 바뀌면 왜 다른 변환이 되는지 설명한다.
  • 큰 행렬을 보면 0의 위치, 대칭성, 대각·삼각·블록 구조부터 표시한 뒤 일반 계산을 줄인다.

Basis: 03. Matrices · Reorganized around operations, inverses, and structured matrices

A matrix is both a rectangular array of numbers and an operator that maps one vector into another. Because AI uses the same notation for batches of data and learned weights, dimension rules and multiplication order describe the model's data flow.

1. Dimensions Reveal Which Operations Are Valid

  • An m × n matrix has m rows and n columns; each column can be viewed as an m-dimensional vector.
  • Addition and subtraction work componentwise only for equal shapes, while scalar multiplication applies the same factor to every entry.
  • AB exists when the number of columns of A equals the number of rows of B; the result has the rows of A and columns of B.

2. Structured Matrices Simplify Computation

MatrixDefinitionInterpretation
Identity IOnes on the main diagonal, zeros elsewherePreserves values under multiplication
Diagonal or triangularZeros outside specified regionsReduces storage and computation
SymmetricAᵀ = APairwise relation is direction-independent
Block matrixSmaller matrices arranged as entriesRepresents subsystems in a large model

3. Products and Transposes Are Order-Sensitive

Each entry of AB is an inner product between one row of A and one column of B. In general AB ≠ BA, so reversing order changes which transformation is applied first. Transposition swaps rows and columns and reverses the internal order of an entire product.

A(m × n)B(n × p) → AB(m × p) · (AB)ᵀ = BᵀAᵀ · AA⁻¹ = A⁻¹A = I

Dimension intuition. Writing the shapes of a data matrix X, a weight matrix W, and output Y reveals whether XW is valid and how many output features it creates. A dimension mismatch often signals an incorrectly designed transformation, not just a typing mistake.

Key Takeaways

  • Matrix operations obey shape rules that express valid input-output relationships in addition to entrywise arithmetic.
  • Matrix multiplication is generally noncommutative but associative, allowing parenthesization to be chosen for computational efficiency.
  • Transpose, symmetry, diagonal, triangular, and block structure reveal both meaning and opportunities for simpler algorithms.

Practical Tips

  • Annotate every factor with its dimensions and verify inner-dimension compatibility before multiplying any entries.
  • Compute AB and BA for one small numerical example, then describe why the reversed order represents a different composition.
  • When facing a large matrix, mark zeros, symmetry, diagonal, triangular, or block structure before applying a general-purpose calculation.
